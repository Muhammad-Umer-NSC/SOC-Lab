## What is VirusTotal?

**VirusTotal** is a free online service owned by Google that analyzes files, URLs, domains, and IP addresses for malicious content. When you submit a file, VirusTotal scans it against **70+ antivirus engines** simultaneously and returns a verdict — telling you exactly how many engines flagged it and what they identified it as.

In a SOC environment, VirusTotal is used as a **threat intelligence enrichment tool**. Instead of manually submitting suspicious files, we integrate it directly with Wazuh so that whenever a new file is detected on a monitored machine, Wazuh automatically checks it against VirusTotal and generates an alert if it's flagged.

This turns our lab from a passive log collector into something that actively identifies threats.

## What are File Hashes and Why Do They Matter?

A **file hash** is a fixed-length string generated by running a file through a hashing algorithm (like MD5, SHA-1, or SHA-256). It acts as a unique fingerprint for that file.

Two key properties make hashes powerful in security:

- **Deterministic** — the same file always produces the same hash, no matter where it is or what it's named
- **Sensitive** — change even a single byte in the file and the hash changes completely

This means:

- A known piece of malware will always have the same hash, regardless of what it's been renamed to
- Security teams and threat intelligence platforms like VirusTotal maintain databases of **known malicious hashes**
- When Wazuh detects a new file via File Integrity Monitoring, it computes the file's hash and sends it to VirusTotal — which checks it against its database instantly

This is why hash-based detection is so effective. An attacker can rename `malware.exe` to `update.exe`, but they can't change its hash without changing the file itself.

## How the Integration Works

Here's the flow once everything is configured:

```
New file appears on Windows 10 VM
        ↓
Wazuh FIM (File Integrity Monitoring) detects it
        ↓
Wazuh computes the file hash
        ↓
Wazuh sends hash to VirusTotal API
        ↓
VirusTotal checks hash against 70+ AV engines
        ↓
Result returned to Wazuh → Alert generated on dashboard
```

All of this happens automatically in the background — no manual submission required.

## Step 2 — Get Your API Key

1. Once logged in, click your **account icon** (top right)
2. Click **API Key**
3. Copy your API key and keep it somewhere safe

> ⚠️ **Never share your API key or commit it to a public GitHub repo.** Anyone with your key can use your VirusTotal quota. Treat it like a password.

## Step 3 — Configure the VirusTotal Integration in Wazuh

Open the Wazuh dashboard in your browser:

```
https://150.1.7.99
```

Navigate to:

**Menu (three horizontal lines)** → **Server Management** → **Settings** → **Edit Configuration**

Scroll to the very bottom of the configuration file. Just above the closing `</ossec_config>` tag, paste the following block:

```xml
  <integration>
    <name>virustotal</name>
    <api_key>YOUR_API_KEY</api_key>
    <group>syscheck</group>
    <alert_format>json</alert_format>
  </integration>
```

> ⚠️ Replace `YOUR_API_KEY` with the actual key you copied from VirusTotal. Indentation matters — paste it exactly as shown above.

**What each line does:**

- `<name>virustotal</name>` — tells Wazuh which integration module to activate
- `<api_key>` — authenticates your requests to the VirusTotal API
- `<group>syscheck</group>` — scopes the integration to File Integrity Monitoring (syscheck) events only, so VirusTotal is only called when a new file is detected — not on every single alert
- `<alert_format>json</alert_format>` — formats the alert data as JSON so Wazuh can parse and display the VirusTotal response correctly in the dashboard

Click **Save** and then click **Restart Manager** to apply the changes.

## Step 4 — What is the EICAR Test File?

The **EICAR test file** is an industry-standard file used specifically to test antivirus and security tools without using real malware. It's a small, harmless text file that every major antivirus engine and threat intelligence platform — including VirusTotal — recognizes and flags as a test threat.

It exists so that security professionals can safely verify their detection pipeline is working correctly without putting their systems at risk.

> ⚠️ **Important:** Even though the EICAR file is harmless by design, it is still detected as a virus by security tools. Only download and open it inside your Windows 10 VM — never on your host machine. Delete it from the VM once you're done testing.

---

## Step 5 — Disable Windows Defender on the Agent

Before downloading the EICAR file, disable Windows Defender on the Windows 10 VM so it doesn't quarantine the file before Wazuh can detect it:

1. Inside the Windows 10 VM, open **Windows Security**
2. Go to **Virus & threat protection**
3. Under **Virus & threat protection settings**, click **Manage settings**
4. Turn off all protections:
   - Real-time protection
   - Cloud-delivered protection
   - Automatic sample submission
   - Tamper protection

---

## Step 6 — Download the EICAR Test File

Inside the Windows 10 VM, open a browser and search for:

```
eicar test file download
```

Download the **eicar.com.txt** file and save it to your **Downloads** folder — this is the directory we configured Wazuh to monitor in Part 2.

Since File Integrity Monitoring is watching the Downloads folder in real time, Wazuh will detect the new file, compute its hash, and send it to VirusTotal immediately.

---

## Step 7 — Verify the Alert in Wazuh

Open the Wazuh dashboard and navigate to:

**Menu** → **Explore** → **Discover**

Add the following filters from the left panel:

- `agent.ip`
- `agent.name`
- `rule.description`

Set your time range to the last **15 minutes** with auto-refresh every **10 seconds**.

You should see a VirusTotal alert appear showing that the EICAR file was flagged — with details on how many engines detected it and what it was classified as.

That's your integration working end to end. 🎉

> Once you've confirmed the alert, go back to the Windows 10 VM and **delete the EICAR file**, then re-enable Windows Defender.
